📘 Common HTTP Status Codes to Know

Code Name Meaning
200✅ OKThe request was successful. Everything worked as expected.
201✅ CreatedA new resource was successfully created (often after POST).
204✅ No ContentRequest succeeded, but no content is returned (common with DELETE).
301🔁 Moved PermanentlyThe resource has a new permanent URL.
302🔁 Found (Temporary)The resource temporarily moved to another URL.
400❌ Bad RequestThe server couldn’t understand the request (e.g., malformed syntax).
401🔒 UnauthorizedYou must log in or provide credentials.
403🔒 ForbiddenYou are not allowed to access the resource (even if logged in).
404❌ Not FoundThe requested resource doesn’t exist.
405❌ Method Not AllowedThe HTTP method (GET, POST, etc.) is not supported for that endpoint.
408🕓 Request TimeoutThe request took too long and was timed out.
500💥 Internal Server ErrorA general server error (the problem is on the server, not your code).
502💥 Bad GatewayServer got an invalid response from another server.
503⚠️ Service UnavailableThe server is overloaded or down for maintenance.
504🕓 Gateway TimeoutServer didn’t respond in time.

🧠 Tips for Students:

  • 2xx = Success
  • 3xx = Redirect
  • 4xx = Client-side error (your request has an issue)
  • 5xx = Server-side error (the server is having issues)

🧪 Want to simulate some of these?

  • https://httpstat.us/404 — returns a 404
  • https://httpstat.us/500 — returns a 500
  • https://httpstat.us/401 — returns a 401
  • ...just change the number in the URL!
